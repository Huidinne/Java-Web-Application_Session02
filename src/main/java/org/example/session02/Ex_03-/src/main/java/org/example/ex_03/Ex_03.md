1.
    Thông báo lỗi đăng nhập chỉ cần tồn tại trong một lần request khi người dùng submit form. Vì vậy nên lưu vào 
Request Scope để sau khi reload hoặc chuyển trang thì thông báo tự biến mất.
    Nếu lưu vào Session Scope thì lỗi sẽ bị giữ lại lâu hơn, dẫn đến việc người dùng đăng nhập lại đúng nhưng vẫn thấy 
thông báo lỗi cũ → gây nhầm lẫn và trải nghiệm không tốt.

2.
   totalViewCount là dữ liệu dùng chung cho toàn bộ hệ thống, tất cả người dùng đều phải thấy cùng một giá trị, nên cần 
lưu vào Application Scope.
Nếu lưu vào Session Scope thì mỗi nhân viên sẽ có một bộ đếm riêng:
-Nhân viên A xem 2 lần → thấy 2
-Nhân viên B mới vào → thấy 0
→ Không phản ánh đúng tổng lượt xem của toàn hệ thống.

3. Race Condition là tình huống nhiều request cùng truy cập và thay đổi một biến chung tại cùng thời điểm, 
dẫn đến kết quả sai lệch.

4. Integer count = (Integer) application.getAttribute("totalViewCount");
   if (count == null) count = 0;
   count++;
   application.setAttribute("totalViewCount", count);

Khi nhiều người truy cập cùng lúc:
2 request cùng đọc count = 5
Cả 2 cùng tăng lên 6
Ghi lại → vẫn là 6 (đáng lẽ phải là 7)
→ Bị mất dữ liệu

5. 
   Cách phòng tránh:dùng synchronized:
   synchronized (application) {
   Integer count = (Integer) application.getAttribute("totalViewCount");
   if (count == null) count = 0;
   count++;
   application.setAttribute("totalViewCount", count);
   }
