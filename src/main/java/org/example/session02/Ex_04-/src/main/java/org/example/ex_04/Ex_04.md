1. XSS là gì?
XSS (Cross-Site Scripting) là lỗ hổng bảo mật cho phép kẻ tấn công chèn mã JavaScript vào trang web. Khi người dùng truy cập, 
đoạn mã này sẽ được thực thi trên trình duyệt, có thể gây đánh cắp dữ liệu hoặc thay đổi nội dung trang.
Ví dụ input: <script>alert(1)</script>

2. Vì sao <c:out> an toàn hơn ${keyword}?
${keyword} sẽ in trực tiếp dữ liệu ra HTML mà không xử lý, nên nếu dữ liệu chứa script thì trình duyệt sẽ thực thi.
Ngược lại, <c:out value="${keyword}" /> sẽ tự động escape các ký tự đặc biệt như <, >, giúp hiển thị dưới dạng text thay vì thực thi.
So sánh output:
Dùng ${keyword}:<script>alert(1)</script>
→ Trình duyệt sẽ chạy script
Dùng <c:out>:
&lt;script&gt;alert(1)&lt;/script&gt;
→ Chỉ hiển thị chữ, không chạy
Vì vậy <c:out> giúp chống XSS.

3. Sự khác nhau giữa <c:if> và <c:choose>
   <c:if> dùng cho điều kiện đơn lẻ, chỉ kiểm tra một trường hợp.
   <c:choose> dùng cho nhiều điều kiện, giống như if-else if-else trong Java.
Trong bài này, phần “Giá vé” và “Vé còn lại” có nhiều trường hợp khác nhau nên cần dùng <c:choose> để xử lý rõ ràng từng trường hợp.

4. Vai trò của <c:url>
<c:url> giúp tạo URL đúng với context path của ứng dụng.
Ví dụ:<c:url value="/events/1/book"/>
Nếu ứng dụng chạy với context path:
/ticketing
Thì URL thực tế sẽ là:
/ticketing/events/1/book
Nếu dùng hardcode: href="/events/1/book"
Thì khi deploy với context path khác, đường dẫn sẽ bị sai và không truy cập được.
